## Violations Fixed During Audit

All violations were **serious** and required fixes before commit.

### 1. `color-contrast` — Sidebar active nav text

- **Pages**: /dashboard, /tasks, /projects
- **Elements**: `<span>` text inside active `<Link>` nav items
- **Root cause**: TanStack Router appends `activeProps.className` to base `className`. Both `text-text-secondary` and `text-primary-600` were present; Tailwind CSS ordering caused `text-text-secondary` to win, yielding 4.29:1 contrast against the `bg-primary-50` active background.
- **Fix**: Moved `text-text-secondary` and dark variants from base `className` to `inactiveProps.className` so active and inactive states use mutually exclusive color classes.

### 2. `color-contrast` — Activity feed `<time>` elements

- **Pages**: /dashboard
- **Elements**: `<time>` timestamps in activity list items
- **Root cause**: `text-text-secondary` (`oklch(0.55 0.02 264)`) yields ~4.1:1 contrast against the page background at `text-xs` (12px). Additionally, the stagger animation causes partially-transparent items during scan — amplifying the perceived contrast failure.
- **Fix**: Added `--color-text-muted: oklch(0.42 0.02 264)` token (~6.8:1 contrast) and applied it to `<time>` elements. Updated a11y spec to `waitForTimeout(800)` so axe scans fully-rendered content.

### 3. `color-contrast` — Status badge text (`success` variant)

- **Pages**: /projects
- **Elements**: `<Badge variant="success">Active</Badge>`
- **Root cause**: `text-status-success` (`oklch(0.65 0.17 162)`) on `bg-status-success/10` yielded only 2.62:1 contrast — far below the 4.5:1 requirement.
- **Fix**: Added `--color-status-*-fg` dark tokens for each status (success, error, warning, info) and updated `Badge` component to use them in light mode. Dark mode continues to use the original status colors against dark surfaces where contrast is sufficient.

### 4. `color-contrast` — Project filter tab inactive state

- **Pages**: /projects
- **Elements**: "archived" filter button
- **Root cause**: `text-text-secondary` on `bg-surface-alt` (#f3f5f9) yielded 4.42:1 — just below the 4.5:1 threshold.
- **Fix**: Changed inactive filter buttons to use `text-text-muted` instead of `text-text-secondary`.

### 5. `nested-interactive` — ProjectCard options button

- **Pages**: /projects
- **Elements**: `<article role="button">` containing `<button aria-label="Project options">`
- **Root cause**: WCAG 4.1.2 prohibits focusable descendants inside interactive controls. The `role="button"` on `<article>` made it interactive, and the options `<button>` inside created an illegal nesting.
- **Fix**: Removed `role="button"` / `tabIndex` / `onKeyDown` from `<article>`. Replaced with an invisible overlay `<button class="absolute inset-0">` for the card click, keeping the options button as a sibling at `z-10`.
